Hooks

In order to separate out style from behaviour please add the data-behaviour attribute to your HTML to hoook up event listeners or the data-element attribute to target specific elements in your Javascript. Using this method tells other developers that an element has JS functionality bound to it and avoids things breaking if classes end up being removed.

<button data-behaviour="toggle-navigation">Toggle Navigation</button>

<nav data-element="navigation">
...
</nav>

Targeting elements using these attributes can be long winded, so we have created some helper functions, which can be found in the config file, to keep things short and simple.

config.js

var RN = window.RN || {};

function byBehaviour(selector) {
  return $('[data-behaviour="' + selector + '"]');
}

function byElement(selector) {
  return $('[data-element="' + selector + '"]');
}

function dataString(attribute, selector) {
  return '[data-' + attribute + '="' + selector + '"]';
}

The Revealing Module Pattern

In order to compartmentalise code, you should make use of the revealing module pattern, namespace your modules to RN to ensure consistant practice across all projects e.g. RN.moduleName... and keep your init function for module config and event listeners and try to keep event listeners to one line where possible.

Keep your init function at the top of the module so other developers can quickly see what the module does.

RN.moduleName = function() {

  'use strict';

  function init() {
    byBehaviour('toggle-navigation').on('click', toggleNavigation);
  }

  function toggleNavigation() {
    byElement('navigation').toggleClass('navigation--is-open');
  }

  return {
    init: init
  };

}();

Handling Markup

Sometimes you may need to inject some markup onto the page. Storing the HTML template as a string in you Javascript is a bad idea as it can make it difficult to find for other developers who aren't familiar with the project, as well as making it hard to read. If you need to inject markup on to the page, then store that makrup in a handlebars template on the page, that way other developers will be able to see that there is markup being dynamically injected on to the page.

Tooling

Grunt

When working on a non Rails project, we use Grunt as our build tool, this basically does all the things Rails asset pipeline does. When picking up a Grunt based project, there are a couple of things you need to do to get up and running:

  • Clone the repo
  • Run npm install
  • Run grunt

What grunt does:

  • Compiles Sass to CSS
  • Concatinates & Uglifies Javascript
  • Compiles HTML templates (optional)
  • Minifies SVGs
  • Optimises Images
  • Starts a local server (localhost:3000)
  • Watches for any other changes

Live Reload

The watch tasks listens out for any changes and re-compiles the appropriate files when you save. If you have the Live Reload plugin installed in your browser and turned on then it will autmatically refresh the page too.

3rd Party Libraries

For all our projects we make use of several third party libraries, these are:

  • jQuery - v2.x hosted on Google CDN
  • Modernizr - feature detection library
  • ParsleyJS - form validation library

3rd Party Plugins

For projects which need advanced functionality, there are often plugins we can utilise to save time. We recommend you use a plugin if there is one available above writing your own as:

  • It will save time
  • We won't need to maintain it
  • Someone else is maintaining it
  • Has been tested thoroughly by lots of different people
  • Has a community to lean on for support

Here is a list of commonly used plugins:

  • FancyBox - multi-purpose lightbox
  • FancySelect - custom select box
  • Slick Slider - image/content slider
  • Pickadate - custom date/time picker
  • Sundial - an alternative to pickadate, allows date/time picker in one input
  • NoUiSlider - an awesome range slider plugin
  • Preloader - a really good article on preloading images

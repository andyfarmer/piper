HTML

Syntax

  • Use soft tabs with two spaces for Rails, or four spaces for PHP â€” they're the only way to guarantee code renders the same in any environment
  • Always use double quotes for attribute values
  • Use whitespace to define logical blocks of code
  • Include a trailing slash in self-closing elements
  • Always use sentence casing for all text within HTML and use CSS to apply lowercasing, or uppercasing to meet design requirements
  • Please seperate out re-usable code into partials where possible/appropriate

Document Setup

Doctype

Enforce standards mode and more consistent rendering in every browser possible with this simple doctype at the beginning of every HTML page.

<!DOCTYPE html>
<html>
  <head>
  </head>
</html>

Language Attribute

Authors are encouraged to specify a lang attribute on the root html element, giving the document's language. This aids speech synthesis tools to determine what pronunciations to use, translation tools to determine what rules to use, and so forth.

<html lang="en">
  <!-- ... -->
</html>

IE Compatability Mode

Internet Explorer supports the use of a document compatibility tag to specify what version of IE the page should be rendered as. Unless circumstances require otherwise, it's most useful to instruct IE to use the latest supported mode with edge mode.

<meta http-equiv="X-UA-Compatible" content="IE=Edge">

Character Encoding

Quickly and easily ensure proper rendering of your content by declaring an explicit character encoding. When doing so, you may avoid using character entities in your HTML, provided their encoding matches that of the document (generally UTF-8).

<meta charset="UTF-8">

Mobile Meta Tags

Include the following meta tags to deal with scaling on mobile browsers.

<meta name="viewport" content="width=device-width initial-scale=1.0">

Boolean Attributes

Do not add a value to boolean attributes. The presence of a boolean attribute on an element represents the true value, and the absence of the attribute represents the false value.

<input type="text" disabled>
<input type="checkbox" checked>

Reducing Markup

Whenever possible, avoid superfluous parent elements when writing HTML. Many times this requires iteration and refactoring, but produces less HTML. Take the following example:

<!-- Not so great -->
<span class="avatar">
   <img src="...">
</span>
  
<!-- Better -->
<img class="avatar" src="...">

Semantics

Markup should be as semantic as possible and presentational markup should be avoided however it is understood that to make something look awesome, this is sometimes unavoidable. Be aware of the document outline produced by your markup, and ensure it makes sense, there are browser extensions to aid in visualising your outline.

Use HTML5 elements, but make sure you are using them appropritatley and in the right way, HTML5 Doctor is a gret resource for checking how elements should be used.

Validation

Code should always be tested against the W3C, to ensure the markup is well formed, however it is understood that 100% valid code is not always possible.

Hooks

  • Always use classes for styling hooks
  • For javascript hooks use data-behaviour for hooking up event listeners, and data-element for targeting specific elements in the DOM
